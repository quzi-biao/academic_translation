# Security Policy

## Supported Versions

当前仓库以 `main` 分支的最新状态为主要维护对象。

| Version | Supported |
| --- | --- |
| `main` | Yes |
| 历史提交 / 旧分支 | No |

## Reporting a Vulnerability

如果你发现了安全问题，请不要直接公开提交带有敏感细节的 Issue、Discussion 或 Pull Request。

建议通过以下方式私下联系：

- 小红书：[https://xhslink.com/m/364W9UKcVdZ](https://xhslink.com/m/364W9UKcVdZ)
- 微信沟通方式可参考仓库 `README` 中的商务合作部分

为了帮助我们更快定位问题，建议在反馈中尽量包含以下信息：

- 漏洞类型
- 影响范围
- 复现步骤
- 可能的利用条件
- 如有需要，可附最小 PoC

## Response Process

收到安全问题后，我们会尽量按以下流程处理：

1. 确认问题是否成立
2. 评估影响范围与优先级
3. 制定修复方案
4. 在修复完成后更新仓库
5. 如有必要，补充公开说明

## Disclosure Guidance

在问题修复前，请尽量不要公开：

- 可直接复现的攻击步骤
- 真实密钥、证书、连接串、部署地址
- 可能影响线上系统安全的内部实现细节

如果你不确定某个发现是否属于安全问题，也欢迎先私下沟通。
